Software Development

The programming language this study guide is focusing on will be Java, so the content here will only be relevant for that. Every programming language is different and has their strengths/weaknesses, Java's strength is it's popularity in desktop and mobile general use software. Don't get this wrong though, most stuff you pick up from here translates really well to other object oriented programming languages! To get started, you'll first want to make sure you're using an IDE (Integrated Development Environment) designed for use with Java. An IDE is basically where all you code can be understood and processed, with many features to assist in your programming and debugging. Without an IDE, programming would be significantly more difficult, let's thank technology!

I will be using NetBeans IDE, and I suggest you do too, at least for his study guide, as this is typically the most commonly used in the industry.

Good practice

When coding in Java, and any other programming language, you want to make sure you're implementing good practice within your code.

What is good practice? Here's the breakdown:
  • Using proper indentation in your code
  • Use short but descriptive names for variables
  • Use methods wherever you have repetitive code
  • Use optional extensions (ex. Prettier) to ensure your code is the best it can be
  • Only use global variables wherever it is necessary

    Variables

    Variables are a fairly simple concept to grasp, they're a core piece of the functionality in programming.

    In Java, there are 5 types of variables, these include: String, Integer, Char, Float, and Boolean.

    Strings hold text, for example: "Hello World". Integers hold numbers, for example: 57. Chars hold a single character, for example "A". Floats hold floating point numbers, for example: 20.05. Booleans hold 2 things, either true or false.

    To help you visualize what variables are and how they're used, let's imagine them in a real world scenario.

    I can have a variable called "Animal", and I can give it the string "Cat".

    To explain what happened here, I have a box called animal, and I can put the cat in that box, now whenever you reference that box, the cat will be part of it.

    This is what this would look like in actual Java!:

    String Animal = "Cat";

    Try to make your own scenarios, just remember that the variable is the container, what you designate to be inside of it should be relevant to the variable name.

    Comments

    Comments are used to describe functionality of code without causing the interpreter to react to the text you have written. These are crucial for clear and readable code, especially in teamwork scenarios. to comment a single line of code, you can simply type "//" before any text you wish to input.

    //Hello world!

    to comment multiple lines of code, you can type "/*" at the start of the text and "*/" at the end!

    /*Hello world!
    This is a multi
    line comment!*/

    Check this out!

    Loops

    In java, there are there are some cases where you'll need to be using loops, there will make a cycle that can only be broken with certain factors, like if a number becomes more than another or if something changes to true.

    While loops are set to run while a variable is in expected values, until it falls out of this expected value which causes the loop to break.

    For example, in a while loop you can use numbers, so while a number is within a value bracket you specify (such as under 5), the loop will continue. Another way to do this is to use Booleans, so that while something is true, the loop will continue.

    White Box Testing and Black Box Testing

    When writing code, you want to make sure it works for its intended purpose, there are many methods you could use to test it, but most commonly (and relevant to this HNC), you'll see white box and black box testing.

    White box testing involves testing the program by individual pieces of code, you know what you're trying to test and focus on it. Imagine it like this: The box is white, so you can see into it and all of the working components of it.

    Black box testing involves testing the program without focusing on individual pieces of code in mind, you just want to make sure it fully works on the outside when a normal person is using it. Imagine it like this: The box is black, so you can't see into it, you can only see then outer shape and it's general functionality.
