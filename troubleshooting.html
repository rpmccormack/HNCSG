Troubleshooting

The key things to know about troubleshooting for the exam is the stages of troubleshooting and the documentation used. You should be able to write about a time you investigated, solved and tested a computer problem.

Investigation stage

Define the problem, gather information, check for any recent changes. Documents used:
  • Error logs
  • Help desk tickets
  • Problem description
    •
    Solution stage
    Make a plan and implement a solution to the problem. Document the process with:
  • A list of possible solutions
  • Step-by-step guide of the solution
    •
    Take note of the solution that works.

    Testing Stage Review and evaluate the steps taken to resolve the issue. Produce:
  • Test plans
  • Test logs
  • Analyse and document the effects
  • Final notes


    • The benefit of documenting the troubleshooting process is that it saves time in the future if you or a colleague encounter the same issue. It would help to store this documentation in a database that all technicians in the organisation could access as required.
